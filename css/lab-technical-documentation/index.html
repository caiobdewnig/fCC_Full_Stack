Regex

Regex, ou expressão regular (do inglês regular expression), é uma ferramenta poderosa usada para buscar, combinar e manipular padrões em textos. Ela funciona como uma linguagem própria, composta por símbolos e regras que descrevem sequências de caracteres.

Imagine que você quer encontrar todos os números de telefone em um documento, ou validar se um e-mail está no formato correto. Fazer isso manualmente ou com código comum pode ser trabalhoso — mas com uma expressão regular bem escrita, você resolve isso com uma única linha.

Exemplos práticos de uso: Verificar se um CPF tem o formato 000.000.000-00

Extrair todos os links de uma página HTML, validar senhas com regras específicas (mínimo de 8 caracteres, com letras maiúsculas e símbolos, por exemplo), e dividir um texto em frases, palavras ou linhas.

Sintaxe básica:

  • . → qualquer caractere (exceto nova linha)
  • \d → um dígito (0–9)
  • \w → um caractere alfanumérico
  • * → zero ou mais ocorrências
  • + → uma ou mais ocorrências

Exemplo: A regex ^\d{2}/\d{2}/\d{4}$ valida uma data no formato dd/mm/aaaa, como 04/08/2025.

DOM

DOM significa Document Object Model, ou Modelo de Objeto de Documento. É a forma como os navegadores representam uma página web internamente. Quando você abre um site, o navegador transforma o HTML da página em uma estrutura de objetos que podem ser acessados e modificados com JavaScript — isso é o DOM.

Pense no DOM como uma árvore, onde cada parte do site (como div, h1, p, etc.) vira um nó (ou node). Com JavaScript, você pode usar o DOM para mudar textos, trocar estilos, responder a cliques, criar elementos novos, remover partes da página, entre muitas outras ações.

Exemplo prático: Se uma página tem esse HTML:

<p id="mensagem">Olá!</p>

Você pode mudar o texto com JavaScript assim:

document.getElementById("mensagem").textContent = "Oi, tudo bem?";

Com o DOM, você pode:

  • Interagir com elementos da página em tempo real
  • Criar interfaces dinâmicas (sem recarregar a página)
  • Fazer validação de formulários
  • Criar animações, efeitos e interações com o usuário
  • Manipular eventos como cliques, teclado e mouse
JavaScript Functions

As funções em JavaScript são blocos de código reutilizáveis que executam uma tarefa específica. Elas são fundamentais para organizar e estruturar seu código, evitando repetição e tornando-o mais legível e maintível.

Uma função pode receber parâmetros (dados de entrada), processar esses dados e retornar um resultado. Existem várias formas de declarar funções em JavaScript:

Declaração tradicional: function nomeDaFuncao() { }

Expressão de função: const minhaFuncao = function() { }

Arrow function (ES6): const minhaFuncao = () => { }

Exemplo prático de uma função que calcula a área de um retângulo:

function calcularArea(largura, altura) { return largura * altura; }

  • Parâmetros são as variáveis que a função recebe
  • O return define o valor que a função devolve
  • Funções podem ser chamadas quantas vezes precisar
  • Ajudam a evitar código duplicado
CSS Flexbox

Flexbox é um método de layout em CSS que oferece uma maneira eficiente de organizar, distribuir e alinhar elementos em um container, mesmo quando o tamanho dos elementos é desconhecido ou dinâmico.

Para usar Flexbox, você define display: flex no elemento pai (container). Isso transforma os elementos filhos em flex items, que podem ser facilmente controlados com propriedades específicas.

Principais propriedades do container:

  • justify-content: alinha itens no eixo principal
  • align-items: alinha itens no eixo transversal
  • flex-direction: define a direção dos itens (row, column)
  • flex-wrap: permite quebra de linha dos itens
  • gap: define espaçamento entre os itens

Exemplo básico: .container { display: flex; justify-content: center; align-items: center; }

Flexbox resolve muitos problemas comuns de CSS, como centralização vertical, distribuição igual de espaço entre elementos, e criação de layouts responsivos sem cálculos complexos.

API REST

API REST (Representational State Transfer) é um estilo arquitetural para desenvolvimento de serviços web. Uma API REST permite que diferentes aplicações se comuniquem através de requisições HTTP, trocando dados geralmente em formato JSON.

REST usa os métodos HTTP padrão para diferentes operações:

  • GET: buscar/ler dados
  • POST: criar novos dados
  • PUT: atualizar dados existentes
  • DELETE: remover dados

Exemplo de endpoint REST: GET /api/users/123 retorna informações do usuário com ID 123.

Características importantes das APIs REST:

São stateless (não mantém estado entre requisições), usam URLs padronizadas para identificar recursos, retornam códigos de status HTTP apropriados (200, 404, 500, etc.), e podem ser consumidas por qualquer linguagem de programação.

APIs REST são amplamente utilizadas para integrar sistemas, criar aplicações móveis, desenvolver SPAs (Single Page Applications) e permitir comunicação entre microserviços.
